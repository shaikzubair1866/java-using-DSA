1. What is Java?

Answer:
Java is a high-level, object-oriented, platform-independent programming language used to develop applications.

2. Why is Java called a Platform-Independent Language?

Answer:
Java code is compiled into bytecode, which can run on any system that has a JVM.

Example:
Write once, run anywhere (WORA).

3. What is JVM?

Answer:
JVM (Java Virtual Machine) is responsible for executing Java bytecode and converting it into machine code.

Functions:

Loads bytecode
Verifies bytecode
Executes the program
4. What is JRE?

Answer:
JRE (Java Runtime Environment) provides the environment required to run Java programs.

Contains:

JVM
Libraries
Supporting files
5. What is JDK?

Answer:
JDK (Java Development Kit) is used to develop Java applications.

Contains:

JRE
Compiler (javac)
Debugging tools
6. Difference Between JDK, JRE, and JVM
JDK	JRE	JVM
Development Kit	Runtime Environment	Virtual Machine
Compiles & Runs Programs	Runs Programs	Executes Bytecode
7. Why is Java a High-Level Language?

Answer:
Because it is easy to read, write, and understand, and it hides hardware-level details from programmers.

8. What are the Features of Java?

Answer:

Simple
Object-Oriented
Platform Independent
Secure
Robust
Portable
Multithreaded
9. What is Bytecode?

Answer:
Bytecode is the intermediate code generated by the Java compiler. JVM executes this bytecode.

10. What is the Difference Between C++ and Java?
Java	C++
Platform Independent	Platform Dependent
No Pointers	Supports Pointers
Automatic Garbage Collection	Manual Memory Management
11. What is an Object?

Answer:
An object is an instance of a class that contains data and methods.

Example:

class Student {
    int id;
}
Student s = new Student();
12. What is a Class?

Answer:
A class is a blueprint for creating objects.

13. What is Constructor?

Answer:
A constructor is a special method used to initialize objects.

class Student {
    Student() {
        System.out.println("Constructor Called");
    }
}
14. What is Method Overloading?

Answer:
Defining multiple methods with the same name but different parameters.

int add(int a, int b) { }
int add(int a, int b, int c) { }
15. What is Method Overriding?

Answer:
A subclass provides a specific implementation of a method already defined in the parent class.

16. What is Inheritance?

Answer:
Inheritance allows one class to acquire properties and methods of another class.

class A { }
class B extends A { }
17. What is Polymorphism?

Answer:
One method can perform different tasks based on the object.

Types:

Compile-time (Overloading)
Run-time (Overriding)
18. What is Encapsulation?

Answer:
Wrapping data and methods into a single unit (class).

Use private variables and public methods.

19. What is Abstraction?

Answer:
Hiding implementation details and showing only essential features.

Achieved using:

Abstract classes
Interfaces
20. What is Garbage Collection?

Answer:
Java automatically removes unused objects from memory to free space.
